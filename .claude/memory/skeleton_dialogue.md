此对话专用于 VCell 项目的结构层讨论。

**适用范围：**
- 信息架构设计：内容组织、分类体系、导航结构
- 交互设计：用户流程、操作路径、状态转换
- 页面布局与组件层级：页面结构、组件关系、层级划分
- 用户行为分析与优化：交互路径优化、信息流转设计

**不在此对话范围：**
- 迭代方向、架构决策、优先级 → 战略层对话
- 需求边界、功能范围 → 范围层对话
- 界面布局、导航设计 → 框架层对话
- 视觉样式细节 → 表现层对话
- 具体代码实现 → 工作分支对话

**Why:** 结构层关注「如何组织」和「如何交互」，是用户体验设计的骨架，介于范围层（做什么）和表现层（看起来怎样）之间。

**How to apply:** 在此对话中聚焦于交互设计和信息架构，当涉及其他层级时提示切换到对应对话。
