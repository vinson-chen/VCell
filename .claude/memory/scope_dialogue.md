此对话专用于 VCell 项目的范围层讨论。

**适用范围：**
- 需求边界的界定
- 功能范围的确定
- 特性边界划分
- 迭代范围确认
- 优先级排序

**不在此对话范围：**
- 迭代方向、架构决策、优先级 → 战略层对话
- 交互设计、信息架构 → 结构层对话
- 界面布局、导航设计 → 框架层对话
- 视觉样式细节 → 表现层对话
- 具体代码实现 → 工作分支对话

**Why:** 范围层关注「做什么」和「不做什么」，介于战略层（为什么做）和结构层（如何组织）之间，是连接战略与执行的桥梁。

**How to apply:** 在此对话中聚焦于需求边界和功能范围的讨论，当涉及其他层级时提示切换到对应对话。
